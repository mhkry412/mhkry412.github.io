I'm a software engineer with a passion for building clean, functional,
and maintainable systems. I believe that good software should feel
inevitable -- obvious in hindsight, invisible in use.

My approach is rooted in simplicity. I favor clear architectures over
clever hacks, readable code over terse one-liners, and solutions that
solve real problems over those that chase trends.

### What I do

I work across the full stack, from designing database schemas and APIs
to crafting responsive interfaces. I've built everything from
real-time collaboration tools to monitoring infrastructure to static
site generators.

### Outside of code

When I'm not programming, you'll find me exploring type theory,
reading about distributed systems, or experimenting with digital
audio processing. I'm also a firm believer in the [Dead Simple
Sites](https://deadsimplesites.com/) philosophy: less, but better.

Let's build something great together.
